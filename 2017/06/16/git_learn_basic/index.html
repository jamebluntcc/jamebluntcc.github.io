Git

git 主要的概念

git主要分为工作区,暂存区和保存区 文件主要以跟踪与未跟踪两种存在形式;
git的保存区又可以被理解为一个版本,或者称为一个快照;

git init将当前操作目录变成Git可以管理的一个仓库。
判断一个目录是不是一个管理仓库，在当前仓库下是否存在一个.git/的文件夹，linux下使用ls -ah查看。
添加文件到仓库：

  • 查看工作区的状态:git status(得出文件是否被修过)
  • 查看具体修改:git diff
  • git log 显示从最近到最远的提交日志
  • git reset –hard HEAD^ 表示回溯到上一个commit版本
    • HEAD表示当前版本 HEAD^ 表示上一个版本，同理HEAD^^

  • git relog查看命令历史，以便确定回到未来哪个版本

  • 第一步是用git add把文件添加进去，实际上就是把文件修改添加到暂存区；

  • 第二步是用git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支；
  • 当你改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令git checkout – file；
  • 当你不但改乱了工作区某个文件的内容，还添加到了暂存区时，想丢弃修改，分两步，第一步用命令git reset HEAD file，就回到了场景1，第二步按场景1操作。

管理分支

git 常见使用命令集

  • git init:将一个项目文件夹建立为git管理的代码仓;
  • git status:查看git管理的文件变动,可以使用-s参数查看简略概要;
  • git add:将工作区的文件保存到暂存区;
  • git commit -m :将暂存区的文件保存进保存区,并且添加备注;
  • git checkout – :将工作区的改动撤销;
  • git rest HEAD :HEAD表示当前保存区的版本,将暂存区的文件撤销到工作区;
  • git diff:查看具体的变动,其中参数–staged可以查看到已经暂存区和保存区的差异,而不加任何参数的git diff默认看到工作区和暂存区的变化;
  • git rm:删除文件,准确地讲应该是删除在暂存区里的文件,所以在git仓下的文件直接删除后还可以找回;
  • git log:查看历史提交,常用命令:git log –pretty=oneline;
  • git remote:查看远程仓库,参数-v查看具体的信息;
    • git add 添加一个新的远程git仓库，同时指定一个简写;
    • git rename 更改远程仓的nickname;
  • git tag:给提交打上标签
    • 参数-a 表示注释型,可以被校验的,其中包括打标签者的名字,email,date等完整信息;
    • 不加参数表示默认轻量标签,只是一个特定提交的引用;

git 分支管理

  • git branch:查看当前全部分支;
    • git branch 新建分支;
    • git branch -d 删除分支;
  • git checkout 切换到分支;
    • git checkout -b 新建并切换到dev分支;
  • git merege 将 合并到当前分支;
    • git branch –merged:查看哪些分支已经合并到当前分支;
    • git branch –no-merged:查看没有合并的工作分支;

tips

  • 删除一个已经修改过并且已经放到暂存区的文件,最好使用git rm -f(force)这时一种安全特性,用于防止失误删除还未添加到快照的数据;
  • 将文件从git暂存区删掉,但是希望保留到当前目录(工作区)时使用git rm –cached;

  • 建立.gitignore文件的时候,如果文件已经事先被track,那么新加到.gitingore的内容会失效,所以需要将文件在暂存区删除掉再track:

  • 后期标签:对以前的提交打标签可以先git log –pretty=oneline 调出全部的提交版本,然后再git tag -a -m ‘’ xxx(版本号);
